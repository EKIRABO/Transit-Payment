# Transit-Payment
#  Smart Transit Payment System

A full-stack digital payment system for public transportation that enables passengers to pay for rides using QR codes and a wallet system, with simulated Mobile Money (MoMo) integration.

---

##  Overview

The Smart Transit Payment System is designed to modernize public transport payments by replacing cash with a secure, digital solution.

Passengers can scan QR codes on buses to pay instantly, while administrators manage routes, buses, and fare systems through a centralized backend.

##  Mobile Money Simulation

Due to limited access to real telecom APIs, the system includes a custom MoMo simulator that mimics real-world behavior:

* Simulates MTN/Airtel payment requests
* Mimics asynchronous payment confirmations (webhooks)
* Ensures realistic transaction flow for testing
* Allows safe development without external dependencies

---

##  QR Code System

The system uses QR technology for seamless boarding:

* Each bus has a unique QR code generated by the admin
* QR codes can be printed and placed inside buses
* Passengers scan the QR code using the frontend
* The system identifies the bus and automatically processes payment

##  Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd Transit-Payment
```

---

### 2. Setup Backend

```bash
cd backend
npm install
```

Create a `.env` file:

```env
DB_HOST=your_host
DB_PORT=5432
DB_USER=your_user
DB_PASSWORD=your_password
DB_NAME=your_db
```

Run the server:

```bash
npm start
```

---

### 3. Setup Frontend

```bash
cd transitpay-frontend
npm install
npm run dev
```

##  API Endpoints (Sample)

### Users

* `POST /users` → Create user
* `GET /users/:id` → Get user profile

### Wallet

* `POST /wallets` → Create wallet
* `GET /wallets/:user_id` → Get balance
* `POST /wallets/topup` → Add funds

### Payments

* `POST /rides/pay` → Pay for a ride via QR

##  Key Design Decisions

* **QR-based payments** to eliminate cash handling
* **Mobile Money simulation** to replicate real-world systems
* **Atomic database transactions** to ensure consistency and prevent payment errors
* **Modular backend architecture** for scalability and maintainability
